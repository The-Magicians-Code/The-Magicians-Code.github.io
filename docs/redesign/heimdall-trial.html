    Problem

    The camera lived on the boat, not in the cloud. A hull-mounted sensor had to name what it saw — buoys, other vessels, the line where haze meets water — from a single-board computer with no network, a strict thermal ceiling, and a battery that resented every wasted watt. Off-the-shelf runtimes leaned on a GPU the vessel would never carry.

    Approach

    I built the inference path in Rust, compiling the model to an ONNX graph and driving it through ONNX Runtime with a hand-tuned execution provider. Frames arrive straight from the kernel over v4l2, skip the usual copy through userspace, and feed a quantised network small enough to hold in cache. A parallel WebAssembly build lets the same pipeline run in a browser for field debugging on a phone at the dock.

    Results

    Steady detection at the sensor's native framerate on a passively cooled board, well under the thermal budget, with headroom to spare for the navigation stack sharing the chip. The WebAssembly twin reproduces the edge output bit-for-bit, so a bug seen at sea can be reproduced on a laptop without the boat. The vessel now sees the sea it sails.
